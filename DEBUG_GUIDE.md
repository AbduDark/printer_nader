# دليل التصحيح (Debug) - تطبيق طباعة الباركود

## الإعدادات المتقدمة

تم إضافة الإعدادات المتقدمة للـ Debug في المجلد `.vscode/`:

### 1. ملف `launch.json` - إعدادات التشغيل والتصحيح

يحتوي على 3 إعدادات:

#### أ) تشغيل تطبيق الطباعة (Debug)
- يبني المشروع تلقائياً قبل التشغيل
- يعمل على إصدار Debug
- استخدمه أثناء التطوير

#### ب) تشغيل تطبيق الطباعة (Release)
- يبني المشروع بإصدار Release
- محسّن للأداء
- استخدمه للاختبار النهائي

#### ج) Attach to Process
- لربط المصحح بعملية قيد التشغيل

### 2. ملف `tasks.json` - المهام

يحتوي على المهام التالية:
- **build**: بناء المشروع (Debug)
- **build-release**: بناء المشروع (Release)
- **clean**: تنظيف ملفات البناء
- **publish**: نشر التطبيق
- **restore**: استعادة الباكجات

### 3. ملف `settings.json` - إعدادات VS Code

يحتوي على:
- تفعيل محلل Roslyn
- دعم EditorConfig
- تنظيم الـ imports تلقائياً
- إخفاء مجلدات bin و obj

## كيفية استخدام الإعدادات المتقدمة

### في VS Code:

1. **فتح لوحة Debug**:
   - اضغط `Ctrl+Shift+D` أو `Cmd+Shift+D` (Mac)
   - أو اذهب إلى View → Run

2. **اختيار إعداد التشغيل**:
   - من القائمة المنسدلة أعلى اللوحة
   - اختر "تشغيل تطبيق الطباعة (Debug)"

3. **بدء التصحيح**:
   - اضغط F5
   - أو اضغط الزر الأخضر "Start Debugging"

4. **وضع نقاط التوقف (Breakpoints)**:
   - اضغط على يسار رقم السطر لإضافة breakpoint
   - سيتوقف البرنامج عند هذه النقطة

5. **أدوات التصحيح**:
   - **Continue** (F5): متابعة التنفيذ
   - **Step Over** (F10): تنفيذ السطر الحالي
   - **Step Into** (F11): الدخول إلى الدالة
   - **Step Out** (Shift+F11): الخروج من الدالة
   - **Restart** (Ctrl+Shift+F5): إعادة التشغيل
   - **Stop** (Shift+F5): إيقاف التصحيح

التطبيق هو Windows Forms Application ويحتاج بيئة Windows للتشغيل الكامل. على Replit (Linux):

   ```bash

   ```

2. **لتشغيل التطبيق**:
   - يحتاج بيئة Windows
   - أو استخدام Wine (دعم محدود)

## البناء والنشر

### بناء إصدار Debug:
```bash
cd printer_nader
dotnet build
```

### بناء إصدار Release:
```bash
cd printer_nader
dotnet build -c Release
```

### نشر التطبيق:
```bash
dotnet publish -c Release -o ../publish
```

سينشئ ملفات التطبيق في مجلد `publish/`

## مشاكل شائعة وحلولها

### المشكلة 1: لا تظهر الإعدادات المتقدمة
**الحل**: 
- تأكد من فتح المجلد الرئيسي للمشروع في VS Code
- تأكد من وجود مجلد `.vscode/` بالملفات الثلاثة

### المشكلة 2: خطأ في البناء
**الحل**:
```bash
cd printer_nader
dotnet restore
dotnet clean
dotnet build
```

### المشكلة 3: التطبيق لا يعمل على Replit
**السبب**: التطبيق مصمم لـ Windows Forms
**البدائل**:
1. استخدم Windows للتشغيل الكامل
2. حوّل التطبيق إلى ASP.NET Core (web-based)
3. حوّل إلى تطبيق Console بدون GUI

## نصائح للتطوير

1. **استخدم IntelliSense**:
   - اضغط `Ctrl+Space` للحصول على اقتراحات الكود

2. **تنسيق الكود**:
   - اضغط `Shift+Alt+F` لتنسيق الملف

3. **البحث في المشروع**:
   - اضغط `Ctrl+Shift+F` للبحث في كل الملفات

4. **الانتقال إلى تعريف**:
   - اضغط `F12` على أي function/class للذهاب إلى تعريفها
